## 🚀 Пошаговое создание .mcaddon

### Шаг 1: Проверка структуры

```

├── items/
│   └── drill_rig.json

```

### Шаг 2: Создание ZIP архива

#### Вариант A: Через файловый менеджер
1. Выделите все файлы и папки мода
2. Правый клик → "Создать архив" или "Отправить" → "Сжатая папка"
3. Назовите архив `drill_rig_mod.zip`

#### Вариант B: Через командную строку (Linux/Mac)
```bash
cd /path/to/drill_rig_mod
zip -r drill_rig_mod.zip . -x "*.git*" "*.DS_Store*" "*.mcaddon"
```

#### Вариант C: Через PowerShell (Windows)
```powershell
Compress-Archive -Path * -DestinationPath drill_rig_mod.zip
```

### Шаг 3: Переименование в .mcaddon
1. Найдите созданный `drill_rig_mod.zip`
2. Переименуйте его в `drill_rig_mod.mcaddon`
3. Убедитесь, что расширение изменилось

### Шаг 4: Проверка файла
- Размер файла должен быть несколько КБ
- Расширение должно быть `.mcaddon`
- Файл должен открываться как архив

## 🔧 Альтернативные способы

### Использование специальных инструментов
1. **Minecraft Bedrock Addon Creator**
2. **Bridge** - редактор модов
3. **Blockbench** - создание ресурсов

### Автоматическая упаковка
```bash
# Скрипт для автоматической упаковки
#!/bin/bash
MOD_NAME="drill_rig_mod"
zip -r "${MOD_NAME}.zip" . -x "*.git*" "*.DS_Store*" "*.mcaddon"
mv "${MOD_NAME}.zip" "${MOD_NAME}.mcaddon"
echo "Мод упакован в ${MOD_NAME}.mcaddon"
```

## 🚨 Решение проблем

### Файл не создается
- Проверьте права доступа к папке
- Убедитесь, что есть свободное место
- Попробуйте другой способ создания архива

### Файл поврежден
- Пересоздайте архив
- Проверьте, что все файлы корректны
- Используйте другой инструмент для архивирования

### Мод не устанавливается
- Проверьте формат файла
- Убедитесь, что структура папок правильная
- Проверьте совместимость версии
