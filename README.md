# Minecraft Bedrock Server на Python

Полностью рабочий сервер для Minecraft Bedrock Edition (Minecraft PE), написанный с нуля на Python 3.11+.

## Возможности

✅ **Поддержка протокола Minecraft Bedrock v1.20.15+**  
✅ **RakNet сетевой протокол** (упрощённая реализация)  
✅ **Отображение в списке серверов** (Unconnected Ping)  
✅ **Подключение игроков** (оффлайн-режим)  
✅ **Генерация плоского мира** (grass, dirt, stone)  
✅ **Перемещение игроков**  
✅ **Размещение и удаление блоков**  
✅ **Чат между игроками**  
✅ **BatchPacket с LZW-сжатием**  
✅ **Многопользовательский режим**  
✅ **Сохранение мира в JSON**  
✅ **Встроенная консоль** с командами  
✅ **Логирование и обработка ошибок**  

## Требования

- **Python 3.11+**
- **Linux/macOS/Windows** (тестировалось на Linux)
- **Порт 19132** (UDP) должен быть открыт

## Установка

1. **Клонируйте репозиторий:**
```bash
git clone <repository-url>
cd minecraft-bedrock-server
```

2. **Убедитесь, что у вас Python 3.11+:**
```bash
python3 --version
```

3. **Запустите сервер:**
```bash
python3 main.py
```

4. **Или запустите демо для просмотра возможностей:**
```bash
python3 demo.py
```

## Подключение к серверу

### Из Minecraft Bedrock (Android/iOS):

1. **Откройте Minecraft Bedrock Edition**
2. **Перейдите в "Игры" → "Серверы"**
3. **Нажмите "Добавить сервер"**
4. **Заполните поля:**
   - **Имя сервера:** Python Bedrock Server
   - **IP-адрес:** `192.168.1.X` (IP вашего компьютера)
   - **Порт:** `19132`
5. **Нажмите "Сохранить"**
6. **Подключитесь к серверу**

### Как найти IP-адрес:

**Linux/macOS:**
```bash
ip addr show
# или
ifconfig
```

**Windows:**
```cmd
ipconfig
```

### Тестирование подключения:

```bash
# Проверьте, что порт открыт
netstat -an | grep 19132

# Или используйте nmap
nmap -p 19132 <your-ip>
```

## Команды консоли

Сервер имеет встроенную консоль с расширенными командами:

### Основные команды:
- `stop` - Остановить сервер
- `say <текст>` - Отправить сообщение всем игрокам
- `kick <ник> [причина]` - Кикнуть игрока
- `list` - Показать список игроков
- `help [команда]` - Показать справку

### Административные команды:
- `time <day|night|noon|midnight|значение>` - Установить время дня
- `weather <clear|rain|thunder>` - Установить погоду
- `gamemode <survival|creative|adventure|spectator> [игрок]` - Изменить режим игры

## Архитектура

### Основные компоненты:

1. **MinecraftServer** (`server.py`) - Главный класс сервера
2. **RakNetServer** (`network/raknet.py`) - Сетевой протокол
3. **World** (`world.py`) - Генерация и хранение мира
4. **Player** (`player.py`) - Управление игроками
5. **PacketHandler** (`packets.py`) - Обработка пакетов

### Сетевой стек:

```
Minecraft Bedrock Client
    ↓
RakNet Protocol (UDP)
    ↓
Connection Management
    ↓
Packet Processing
    ↓
Game Logic
    ↓
World Management
```

## Протокол

Сервер реализует упрощённую версию протокола Minecraft Bedrock:

### Основные пакеты:
- **Unconnected Ping/Pong** - Отображение в списке серверов
- **Login/PlayStatus** - Авторизация игроков
- **StartGame** - Инициализация игры
- **LevelChunk** - Передача чанков мира
- **MovePlayer** - Перемещение игроков
- **UpdateBlock** - Изменение блоков
- **Text** - Чат сообщения
- **BatchPacket** - Пакетная обработка

### RakNet функции:
- ✅ Подключение/отключение
- ✅ Надёжная доставка пакетов
- ✅ ACK/NAK механизм
- ✅ Flow control
- ⚠️ Split packets (упрощённо)

## Генерация мира

Сервер генерирует простой плоский мир:

- **Высота:** 64 блока
- **Слои:** Grass (63), Dirt (62-60), Stone (59-0)
- **Размер чанка:** 16×16×128
- **Биом:** Plains
- **Структуры:** Простые деревья (10% шанс)

## Сохранение данных

Мир сохраняется в файл `world.json`:
- Координаты чанков
- Блоки (только не-air)
- Настройки мира
- Позиция спавна

## Логирование

Сервер ведёт подробные логи:
- `server.log` - Файл логов
- Консольный вывод
- Уровни: INFO, DEBUG, ERROR

## Устранение неполадок

### Сервер не запускается:
```bash
# Проверьте Python версию
python3 --version

# Проверьте права на порт
sudo netstat -tulpn | grep 19132

# Запустите с правами администратора (если нужно)
sudo python3 main.py
```

### Клиент не подключается:
1. Проверьте IP-адрес
2. Убедитесь, что порт 19132 открыт
3. Проверьте файрвол
4. Попробуйте локальный IP (127.0.0.1)

### Проблемы с производительностью:
1. Уменьшите количество игроков
2. Ограничьте радиус загрузки чанков
3. Оптимизируйте генерацию мира
